--------------------------------------------------
1. CREACIÓN DEL PROYECTO Y REPOSITORIO
--------------------------------------------------

Se creó el directorio del proyecto en el equipo local:

mkdir srh-backend-fastapi
cd srh-backend-fastapi


--------------------------------------------------
2. INICIALIZACIÓN DE GIT
--------------------------------------------------

Se inicializó el repositorio Git:

git init

Se verificó el estado del repositorio:

git status


--------------------------------------------------
3. CONFIGURACIÓN DEL REPOSITORIO REMOTO
--------------------------------------------------

Se agregó el repositorio remoto de GitHub:

git remote add origin https://github.com/AndreaMM2025/srh-backend-fastapi-t02-03-grupo-b.git

Se verificó el repositorio remoto:

git remote -v


--------------------------------------------------
4. CREACIÓN DEL ENTORNO VIRTUAL
--------------------------------------------------

Se creó el entorno virtual de Python:

python -m venv venv

Se activó el entorno virtual (Windows):

venv\Scripts\activate


--------------------------------------------------
5. INSTALACIÓN DE DEPENDENCIAS
--------------------------------------------------

Se instalaron las dependencias del proyecto:

pip install fastapi
pip install uvicorn
pip install pydantic[email]


--------------------------------------------------
6. EJECUCIÓN DEL BACKEND
--------------------------------------------------

Se ejecutó el backend utilizando Uvicorn:

uvicorn app.main:app --reload

Se verificó el funcionamiento del sistema en:

http://127.0.0.1:8000
http://127.0.0.1:8000/docs


--------------------------------------------------
7. CONTROL DE VERSIONES – COMMITS INICIALES
--------------------------------------------------

Se agregaron los archivos al área de staging:

git add .

Se realizó el primer commit del proyecto:

git commit -m "feat: api base fastapi funcionando y swagger activo"

Se subieron los cambios al repositorio remoto:

git push --set-upstream origin main


--------------------------------------------------
8. ACTUALIZACIÓN DE DOCUMENTACIÓN
--------------------------------------------------

Se modificó el archivo README.md para documentar el proyecto.

Se agregaron los cambios al staging:

git add README.md

Se realizó el commit de documentación:

git commit -m "docs: actualizar README del proyecto"

Se subieron los cambios a GitHub:

git push


--------------------------------------------------
9. SINCRONIZACIÓN DE CAMBIOS ENTRE INTEGRANTES
--------------------------------------------------

Cuando un integrante realizó cambios desde otro equipo, se sincronizó el
repositorio local con el repositorio remoto mediante:

git pull

En caso de existir cambios locales pendientes, se utilizaron los siguientes
comandos para evitar la pérdida de información:

git stash
git pull
git stash pop


--------------------------------------------------
10. VERIFICACIÓN DEL HISTORIAL DE COMMITS
--------------------------------------------------

Se verificó el historial general de commits del repositorio:

git log

Se verificó el historial de commits del repositorio:

git log --oneline

Este comando permitió visualizar de forma resumida los commits realizados
durante el desarrollo del proyecto.


--------------------------------------------------
11. VERIFICACIÓN DETALLADA DE COMMITS Y AUTORES
--------------------------------------------------

Se verificó la autoría de los commits mediante el comando:

git log --pretty=format:"%h | %an | %ae | %ad | %s" --date=short

Este comando permitió identificar claramente el autor, correo, fecha y mensaje
de cada commit realizado por los integrantes del grupo.


--------------------------------------------------
12. ESTADO FINAL DEL REPOSITORIO
--------------------------------------------------

Se comprobó que el repositorio se encontraba actualizado y sin cambios pendientes:

git status

Resultado esperado:
On branch main
nothing to commit, working tree clean
